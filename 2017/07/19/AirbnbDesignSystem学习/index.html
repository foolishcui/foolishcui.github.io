Airbnb Design System 学习

Jul 19, 2017

推荐阅读：
http://airbnb.design/building-a-visual-language/
http://airbnb.design/the-way-we-build/

在软件的设计与开发过程中，由于快速迭代的需要，我们对产品所提出的设计方案很难建立在一套完整的设计语言与组件库的基础之上，这在未来总会造成隐患。

设计始终是与系统以及如何创造可扩展与可重复的产品息息相关。 一套完整的设计语言，能够避免人们对产品细节产生各种误解，同时有助于创造更协调一致的用户体验，帮助设计师们更有效率的工作。

设计系统是什么

设计系统是一套由共同原则和模式定义的组件的集合。 这套系统允许在设计，开发中使用共同的词汇进行快速迭代。 设计系统的结构简单而连贯，降低了团队的沟通成本。

我们为什么需要设计系统

  • 庞大的设计团队
  • 创造协调一致的用户体验
  • 不同的设计师有不同的设计方案和设计风格
  • 多平台用户体验的一致性
  • 旧有的组件需要迭代，与现有组件保持风格一致

Airbnb 是建立设计系统组件库的方法

1.准备工作

设计要保持多平台的体验的一致性，定义可重复使用的组件。在刚开始时主要关注iOS与Android平台。首先打印出产品中所具有的新设计与旧设计，方便查看设计不一致的地方并开始作出修改。每个设计师把注意力集中在一个页面或一个产品领域进行重设计，解决现有的问题。并在这个过程中建立一些设计原则来引导设计。

统一性：每个组件都是整体的一部分，没有独立的设计风格。
普遍性：Airbnb的用户遍布在全世界，产品与视觉语言应该被容易接受。
形象性：设计要能够充分的体现其功能性。
生动性：动效设计要做到栩栩如生，帮助用户与真实世界建立联系。

一个非常重要的准备工作是把现有产品在不同平台上不同功能的截图放在网站上易于设计师查看。

2.奠定基础

建立一套基本的产品设计风格，包含字体排版，颜色，图标，间距，信息架构。把设计风格统一下来能够帮助设计师在独立探索设计方案的时候有一个统一的方向。在每一天结束的时候review每个人的设计成果，会看到不用的pattern逐渐出现。

3.创建组件

把组件看成一个生态系统的基本元素，它们具有特定的功能与特点，有一组属性定义，可以与其他的组件共存同时也可以独立迭代。

组件库需要不断迭代

每一个组件都由其内部的元素所定义（如标题、文字、图片、图标），以及有时包含一些可选择的元素。这些元素在sketch文件与代码里面都已被定义。比如分隔线是一个组件的基本元素，我们要求每个组件都有一个分隔线，而它是否在组件中展示则需要基于当前的视图逻辑。

随着说创建的组件数量不断增长，我们开始把类似的组件放进一个artboard里面，如 Navigation, Marquees, Content, Image, Marquees, Speciality.

我们为手机（iOS和Android）创建了一套组件，并将其适应平板电脑的尺寸。 平板上的组件与移动设备大部分相同，在技术层面上，代码只需要以两种不同的风格存在。 通过这个系统，组件仅仅是在外观和定位方面不同，类似于响应式设计对Web的工作方式。 然后，设计人员可以使用常用组件设计屏幕，并且可以轻松适应不同的屏幕尺寸以及iOS和Android。

对于平板电脑，我们创建了一些简单的布局概念，例如Focus Views，它将内容集中在页面，模态和2列和网格布局上。

经验教训

1.并非所有组件都需要被平等的创建

对于有些使用非常频繁的组件，比如 table 里面的行和列，需要投入更多的时间去思考得到更强大的pattern，即更有效的解决方案。

2.Sketch文件难以维持

过多的sketch文件难以维持，文件的更新于同步也是问题。

3.缺乏文档记录

这个项目需要我们在紧迫的时间内进行操作，这使我们忽视了一些文档过程。 缺乏彻底的文件造成了一些可以避免的混乱。 就像编码一样，在创建系统时记录系统是最重要的过程。 它必须迟早进行，并且在整个创建过程中进行记录，可以让您获得更平滑的决策。
